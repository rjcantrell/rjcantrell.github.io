The Essay.

The narrative visualization correctly follows the hybrid structure stated in the essay.

My visualization is designed as a martini-glass. The user is led through several non-interactable scenes, before being left at an interactive visualization that allows them to chop away at the underlying data a bit. As in my midterm, we're after that age-old elusive question: What does it take to score? In the National Hockey League, I mean. The first scene sets a scene for general scoring ability across the league over time. The second diverts into the scorers themselves, and finally the third looks at it from the perspective of age and position. In this final visualization, several controls are displayed so that the user can choose for themselves what to plot, and for whom to plot it.

The essay properly discusses the layout and design of the scenes of the narrative visualization.

The essay properly discusses the application of the annotations of the narrative visualization.

Data is highlighted in several ways in this visualization. All of the line charts display tooltips when hovered upon with the mouse. The first scene allows the user to display additional annotations. All these text elements follow a consistent theme.

The essay properly discusses the parameters of the narrative visualization, including describing which parameters control the current state of the narrative visualization.

The third scene shows the best use of parameters. In it, a selection of several statistics can be plotted. The user's selection is driven by a radio-button alongside the viz. Each statistic can be plotted for one, many, or all player positions. This is directly communicated to the user by a list of checkboxes alongside the viz. Interaction with these radio buttons and checkboxes are triggers for the viz to change state. Interactive controls are highlighted to the user with bright colors and bold font.

The essay properly discusses the triggers, including what user events trigger what parameter value changes, and how the viewer is aware of available user events.
